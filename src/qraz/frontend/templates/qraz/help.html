What is QRAZ?

QRAZ provides a simple integration of Hovercraft with GitHub.

How does it work?

You can login to QRAZ with your GitHub account and activate one or more of your repositories that contain hovercraft presentations.

How does QRAZ know what files to convert?

Once you have enabled a repository QRAZ will start monitoring it for pushes, using GitHub webhooks. To tell QRAz which file to convert with hovercraft, you need to create a file called .hovercraft.yml inside your repository. This works similar to services like Travis-CI and their .travis.yml. The content of the file needs to be valid YAML.

Here is an example for .hovercraft.yml: 

04-html:
  source: 04-html.rst
  css:
    - css/campus02.css
  template: default
  auto-console: false
  skip-help: true
  skip-notes: true
  assets:
    - figures
    - videos
    - images
    - examples
04-html-simple:
  source: 04-html.rst
  template: simple
This would build two presentations named 04-html and 04-html-simple using the file 04-html.rst at the root of the repository as input for hovercraft. Each presentation in the YAML file can have several attributes, all of which are optional:
source
The name of the input file for hovercraft relative to the repository root. If this is not specified, QRAZ will try to use a file named <presentation-name>.rst.
css
A list of CSS filenames that should be included in the presentation, relative to the repository root.
template
The hovercraft template to use. If not specified, a value of default is used.
auto-console
auto-console
auto-console
auto-console
